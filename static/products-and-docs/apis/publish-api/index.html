    Publish API

    Overview

    Authentication

    To run a successful API call, you will need to pass your API Key as the query parameter apikey.

    Example: https://app.ticketmaster.com/publish/v2/events?apikey={apikey}

    Root URL

    https://app.ticketmaster.com/publish/{API version}

    Publish An Attractions

    Method: POST

    Description: Since 1.0.0

    /publish/v2/attractions

    Other parameters:

    Parameter In Description Type Default Value Required
    data body Attraction
    Attraction
    show all
    • type (string) - Type of the entity
    • source - Source of the entity, where the entity comes from
      • name (string) - Source's name
      • id (string) - ID of the entity in the source's system
    • names (object) - Names of the entity - multi-lingual fields
    • descriptions (object) - Descriptions of the entity - multi-lingual fields
    • additionalInfos (object) - Additional informations of the entity - multi-lingual fields
    • url (string) - URL of a web site detail page of the entity
    • images (array) - Images of the entity
      • url (string) - Public URL of the image
      • ratio (string) - Aspect ratio of the image
      • width (integer) - Width of the image
      • height (integer) - Height of the image
      • fallback (boolean) - true if the image is not the event's image but a fallbak image
      • domains (array) - Domains on which this image should be displayed
      • attribution (string) - Attribution of the image
    • classifications (array) - Attraction's classifications
      • primary (boolean)
      • segment - A Segment is a primary genre for an entity (Music, Sports, Arts, etc)
        • names (object) - The Name of the classification's level - multi-lingual fields
      • genre - Secondary Genre to further describe an entity (Rock, Classical, Animation, etc)
        • id (string) - The ID of the classification's level
        • names (object) - The Name of the classification's level - multi-lingual fields
      • subGenre - Tertiary Genre for additional detail when describring an entity (Alternative Rock, Ambient Pop, etc)
        • id (string) - The ID of the classification's level
        • names (object) - The Name of the classification's level - multi-lingual fields
      • type - A Type represents a kind or group of people. (Donation, Group, Individual, Merchandise, Event Style, etc)
        • id (string) - The ID of the classification's level
        • names (object) - The Name of the classification's level - multi-lingual fields
      • subType - Secondary Type to further categorize an entity (Band, Choir, Chorus, etc)
        • id (string) - The ID of the classification's level
        • names (object) - The Name of the classification's level - multi-lingual fields
    • relationships (array) - Relationships on the entity, like if the entity is a duplicate of another one
    • references (object) - References of this entity in an other system. Reference is the exact same entity
    • version (integer) - Version of the entity. Version is to avoid updated an entity with an older version
    • active (boolean) - Indicate if the entity is active, inactive entity won't appear in Discovery API
    • test (boolean) - Indicate if this is a test entity, by default test entities won't appear in discovery API
    • discoverable (boolean) - True if the entity is dicoverable in discovery API
    		Yes
    TMPS-Correlation-Id header Unique correlation id to be able to trace the request in our system
    		 String Yes

    Response structure:

    • status (string) - Status of the result
    • missingProperties (object) - List of properties that should be present in your entity to ease its dicovery
    • unknownProperties (object) - List of unknown properties that will be dropped
    • invalidValues (object) - List of invalid values
    • invalidProperties (object) - List of invalid properties
    • ignoredProperties (object) - List properties that are ignored in the validation

    Method: PATCH

    Description: Since 1.0.0

    /publish/v2/attractions/{id}

    URL parameters:

    Parameter Description Type Default Value Required
    id ID of the attraction the patch will be applied String Yes

    Other parameters:

    Parameter In Description Type Default Value Required
    data body Patch to apply
    AugmentationData
    show all

    • relatedEntityId (string) - Id of the entity to apply the augmentation data.
    • source (string) - The source where the augementation data came from

    • score (number) - The confidence (%) level of the accuracy of this augmention data. 100 is the better
    • changes (array) - List of changes to apply to the related entity

      • path (string) - Json path from the root of the document to the place where the change should be applied.
      • value (object) - Data to change. MUST be a valid json object.
    		Yes
    TMPS-Correlation-Id header Unique correlation id to be able to trace the request in our system
    		 String Yes

    Response structure:

    • id (string) - Id of the entity ingested in the discovery api
    • status (string) - Status of the result
    • missingProperties (object) - List of properties that should be present in your entity to ease its dicovery
    • unknownProperties (object) - List of unknown properties that will be dropped
    • invalidValues (object) - List of invalid values
    • invalidProperties (object) - List of invalid properties
    • ignoredProperties (object) - List properties that are ignored in the validation
